## How I work

I like building software with a balance between **technical quality**, **business value** and **execution speed**.

- Understand the real problem before writing code
- Keep architecture simple, clear and scalable
- Build useful software, not just technical demos
- Automate repetitive work
- Improve developer experience
- Create maintainable systems
- Connect business needs with technical execution
- Make technical decisions with practical impact
- Think in products, not only in implementations
